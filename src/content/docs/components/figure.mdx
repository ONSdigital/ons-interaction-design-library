The Figure component standardises the presentation of data visualisations across the ONS website. It ensures figures are consistently labelled, attributed and accessible, regardless of the visualisation type or the page template in which they appear.

Figures help users understand quantitative information through clear visual presentation and appropriate contextual detail.

## Purpose and goals

The Figure component provides a structured and reusable way to display visual data in ONS content. It ensures that each figure includes the supporting information users need to understand the data accurately and in context.

Figures support narrative content by providing clear reference points, such as “Figure 2”. They also provide important context, including scope, source and definitions, helping to improve transparency and trust in published data.

## Variants

The Figure component has two variants.

### Borderless (default)

The borderless variant has no outer container border. It should be used in most cases, particularly where the surrounding layout already provides sufficient separation.

### Bordered

The bordered variant includes a subtle container border. It can be used where additional visual separation improves readability, for example in dense layouts or where multiple figures appear close together.

## When to use this component

Use the Figure component when presenting data through charts, tables, maps or other analytical visualisations.

Do not use this component for decorative images or simple standalone statistics. Use a more appropriate component, such as a statistic or key figure, instead.

## Structure and anatomy

A Figure is made up of the following elements.

### Figure number and title (optional)

The figure number and title are combined into a single element and must be used together.

They provide a clear, concise description of the figure and allow it to be referenced within the surrounding content, for example “Figure 3”.

Use this element when the figure needs a standalone description or when it is referenced elsewhere on the page.

The title should be concise, factual and neutral. It should describe what the visualisation shows, not interpret it.

If a number and title are not needed, this element can be omitted entirely.

### Figure subtitle (optional)

The subtitle provides additional context, such as date ranges, geography or population.

Use a subtitle when this information is needed to clarify what the data represents without overloading the title.

### Visualisation

The Figure component supports a wide range of visual content, including charts, tables, maps, diagrams, images and embedded interactive visualisations.

It does not define how a visualisation should be designed internally. Instead, it provides the surrounding structure, including spacing, captions and metadata, so that different types of visualisation can be presented consistently within the page.

### Data source

The data source identifies where the data originates from, including the organisation name.

### Footnotes (optional)

Footnotes can be used to provide additional clarification or context.

They should be used sparingly and only where necessary. Footnotes are presented using the Detail component to allow users to expand them when needed.

### Download links (optional)

Download links allow users to access the underlying data or related assets.

Only show download links when formats are available. A figure may include a single download option or multiple formats, depending on what is provided.

Download links are presented within an expandable Detail component.

#### Behaviour

- Download links are only shown when available
- A figure may include one or multiple formats
- Do not show unavailable or placeholder options

#### Implementation note

The design system defines the layout, styling and placement of download links. The formats themselves are determined in code and should not be fixed in design.

## Behaviour and usage rules

Figure numbers should run sequentially within a page.

Titles should describe the content of the figure and remain neutral in tone. Subtitles should include key contextual information where needed.

Visualisations must remain clear and legible across screen sizes. Colour should not be the only way of conveying meaning.

Footnotes should be used only where they add necessary clarity.

## Figma implementation guidance

Use the appropriate variant based on the layout and content.

Include only the elements required for each figure. Do not add empty or placeholder content.

Ensure spacing and alignment follow the component definition.
