## Purpose and goals

The Figure component provides a structured and reusable way to display visual data in ONS content.

Figures support narrative content by providing clear reference points, such as "Figure 2". They help to visualise the data and should be used to support the surrounding commentary.

## When to use this component

The Figure component can be used as a "container" for the following components:
* charts
* embeds
* images

Do not use this component for decorative images or simple standalong statistics. Use a more appropriate component such as [Headline Facts and Figures](/components/headlinefactsfigures) instead.

## Structure and anatomy

A Figure is made up of the following elements.

### Figure number (optional)

The figure number allows the content to be referenced within the surrounding narrative, for example "Figure 3".

If a figure number is not needed, this element can be ommited entirely.

### Figure title (optional)

Guidance on figure titles can be found in the ONS Service Manual (link to be provided).

### Figure subtitle (optional)

Guidance on figure subtitles can be found in the ONS Service Manual (link to be provide).

### Visualisation

The Figure component supports a wide range of visual content.

It does not define how a visualisation should be designed internally. Instead, it provides the surrounding structure, including spacing, styling and metadata, so that different types of visualisation can be presented consistently across the website.

### Data source (optional)

The data source indentifies where the data originates from, including the organisation name. View source guidance on the ONS Service Manual (link to be provided).

### Footnotes (optional)

Footnotes can be used to provide additional clarification or context.

Footnotes are presented using the Detail component to allow users to expand them when needed.

* Footnotes are only shown when available and are presented in a numbered list.
* A figure may include one or multiple footnotes within the expandable section.
* Footnotes may sometimes include hyperlinks to related content.

Guidance on footnotes can be found in the ONS Service Manual (link to be provided).

### Download links (optional)

Download links allow users to access the underlying data or related assets.

Download links are presented within an expandable Detail component to allow users to expand them when needed.

#### Behaviour

* Download links are only shown when available
* A figure may include one or multiple formats
* Download links should not be numbered or bulleted
* Do not show unavailable or placeholder options

#### Implementation note

The design system defines the layout, styling and placement of download links. The formats themselves are determined in code and should not be fixed in design.

Guidance on downloads can be found in the ONS Service Manual (link to be provided).

## Behaviour and usage rules

Visualisations must remain clear and legible across screen sizes.

## Figma implementation guidance

Include only the elements required for each figure. Do not add empty or placeholder content.

Ensure spacing and alignment follow the component definition.
